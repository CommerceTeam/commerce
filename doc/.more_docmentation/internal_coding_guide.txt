Commerce internal coding guidelines:

Naming conventions for variables
Product objects:	$prodObj
Category objects:	$catObj
Article objects	:	$artObj
Attribute objects:	$attObj
Attribute value objects: $attValueObj


All Frontend Render Methods should be named : renderXdsfindX, parameters are always:
-	DataObjects
-	subpart name(s) as array ore one subpart 
-	typoscript, optional
-	prefix, optional

All essential debug calls should be switched on / off with one cntral debug variable in each class. ($this->debug = false / true). Other debug calls should be removed

HTML Frontend templates should be coded as nested templates with nested subparts and markers. 

Writing-style:
(uk) email 
(de) E-Mail



Checks:


When changing database fields, always check in EM manager if Database definitions are correct und the changes are correctly installed in the DB. After install the EM should not change database again.
